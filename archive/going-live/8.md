Topic 8: **The "Luxury Studio" UI/UX Overhaul**.
The goal of this module is to pivot the visual language from a "software tool" to a "high-end digital studio." We want to prioritize the creator's media, use sophisticated typography, and provide a premium "unboxing" experience for the client.

---

### **Task 1: Refine Global Color Palette to "Neutral Obsidian"**
- **suggested-assignee**: Frontend
- **priority**: High
- **consideration**: Use cool-toned grays and deep blacks rather than pure black. This prevents the UI from clashing with the colors in the photographers' photos.
- **description**: Update Tailwind configuration to replace high-saturation background colors with a neutral, sophisticated palette.
- **requirements**:
    1. Update `tailwind.config.js` to define `surface-950` as a deep obsidian (e.g., `#0A0A0A`).
    2. Reduce the opacity of ambient glow effects in `Layout.tsx`.
- **acceptance-criteria**:
    - The background feels deep and neutral, making photos "pop" without color interference from the UI.

---

### **Task 2: Implement Serif Typography for Editorial Headings**
- **suggested-assignee**: Frontend
- **priority**: Medium
- **consideration**: Serif fonts (like Playfair Display or Lora) communicate luxury and "Vogue-style" editorial quality.
- **description**: Introduce a secondary font family for titles in Proposals and Public Profiles.
- **requirements**:
    1. Import a Google Font (e.g., *Lora* or *Instrument Serif*).
    2. Apply `font-serif` to project titles and proposal headers.
- **acceptance-criteria**:
    - Main titles in the Proposal view look like magazine headers rather than technical labels.

---

### **Task 3: Implement Masonry Layout for Shared Galleries**
- **suggested-assignee**: Frontend
- **priority**: High
- **consideration**: Professional photographers hate "cropped" grids. Masonry preserves the aspect ratio of both portrait and landscape shots simultaneously.
- **description**: Replace the standard CSS grid in `SharedGallery.tsx` with a masonry-style layout.
- **requirements**:
    1. Use a library like `react-plm-masonry` or a pure CSS `columns` implementation.
    2. Ensure the layout is responsive (1 col on mobile, 3+ on desktop).
- **acceptance-criteria**:
    - Photos of different orientations fit together seamlessly without being forced into squares.

---

### **Task 4: Add Cinematic Cover Image to Proposals**
- **suggested-assignee**: Fullstack
- **priority**: High
- **consideration**: This is the first thing a client sees. It should feel like a custom invitation.
- **description**: Allow creators to pick one "Hero" image that acts as the background/header for the Public Proposal.
- **requirements**:
    1. Backend: Add `CoverImageID` to the `Proposal` model.
    2. Frontend: Display this image as a large (400px+) header with a subtle dark overlay on `PublicProposal.tsx`.
- **acceptance-criteria**:
    - The public proposal looks like a personalized landing page for the client.

---

### **Task 5: Create a Sticky "Upload Manager" Tray**
- **suggested-assignee**: Frontend
- **priority**: Medium
- **consideration**: Standard browsers hide progress. A sticky tray gives the user confidence to keep using the app while files upload in the background.
- **description**: Implement a persistent tray at the bottom-right that tracks current upload progress.
- **requirements**:
    1. Create `UploadTray.tsx`.
    2. Show a progress bar and "X of Y files uploaded."
    3. Use a fixed position so it stays visible while the user navigates other albums.
- **acceptance-criteria**:
    - Starting an upload triggers the tray; closing the tray does not cancel the upload.

---

### **Task 6: Design "Premium" Empty States**
- **suggested-assignee**: Junior
- **priority**: Low
- **consideration**: Blank screens feel broken. Beautiful empty states feel like "New Beginnings."
- **description**: Replace generic text with illustrated empty states for Projects, Inquiries, and Albums.
- **requirements**:
    1. Use high-quality SVG icons and serif sub-text.
    2. Example for Projects: "Your first masterpiece starts here."
- **acceptance-criteria**:
    - No screen in the app shows just a blank white/black space when no data exists.

### **Task 7: Implement Skeleton Screens for Media Grids**
- **suggested-assignee**: Frontend
- **priority**: Medium
- **consideration**: Skeletons reduce perceived latency. Match the masonry shape if possible.
- **description**: Add loading skeletons to `ProjectDetail.tsx` and `SharedGallery.tsx`.
- **requirements**:
    1. Create a `MediaSkeleton.tsx` component.
    2. Show animated pulses in the shape of image cards while the API is loading.
