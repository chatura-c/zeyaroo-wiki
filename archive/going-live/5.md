Topic 5: **Growth Engine & Viral Loops**.
The goal of this module is to ensure that every public interaction (shares, proposals, profiles) acts as a marketing funnel for the platform, optimizing for both human users and AI generative engines.

---

### **Task 1: Implementation of OpenGraph Meta Tags for Shared Galleries**
- **suggested-assignee**: Frontend
- **priority**: High
- **consideration**: Use the gallery’s `cover_image` as the `og:image`. If no cover exists, use a standard Zeyaroo branded card.
- **description**: When a client shares their gallery link on WhatsApp, iMessage, or Facebook, a rich preview card should appear.
- **requirements**:
    1. Update `SharedGallery.tsx` (or the index.html template dynamically).
    2. Add `og:title`, `og:description`, and `og:image` meta tags.
    3. Ensure `twitter:card` tags are also present.
- **acceptance-criteria**:
    - Pasting a gallery link into the [Facebook Debugger](https://developers.facebook.com/tools/debug/) shows the correct project title and image.

---

### **Task 2: JSON-LD Structured Data for Creator Profiles (GEO)**
- **suggested-assignee**: Frontend
- **priority**: Medium
- **consideration**: This helps AI search engines like Perplexity and Gemini index creators correctly by their specialty and location.
- **description**: Add a machine-readable JSON-LD script to public creator profiles using the `ProfessionalService` schema.
- **requirements**:
    1. Map `profile.display_name` to `@name`.
    2. Map `profile.service_categories` to `knowsAbout`.
    3. Map `profile.location` to `address`.
- **acceptance-criteria**:
    - The [Google Rich Results Test](https://search.google.com/test/rich-results) recognizes the page as a "Professional Service" entity.

### **Task 3: Add "Powered by Zeyaroo" Referral Hook**
- **suggested-assignee**: Frontend
- **priority**: High
- **consideration**: This must be subtle and elegant. Do not obstruct the creator's work.
- **description**: Add a small footer to all public-facing pages (Galleries and Proposals) that links back to the Zeyaroo landing page.
- **requirements**:
    1. Add a footer component to `SharedGallery.tsx` and `PublicProposal.tsx`.
    2. Text: "Powered by Zeyaroo — The safe way to deliver media."
    3. Include a UTM parameter: `?utm_source=referral_footer`.
- **acceptance-criteria**:
    - Clicking the footer on a public page successfully redirects to the Zeyaroo homepage.

### **Task 4: Dynamic Dynamic Robots.txt and Sitemap Generator**
- **suggested-assignee**: Backend
- **priority**: Medium
- **consideration**: We only want to index *public* creator profiles, never private projects or galleries.
- **description**: Generate a sitemap that lists all active creator profile URLs to help search engines find them.
- **requirements**:
    1. Create a route `GET /sitemap.xml`.
    2. Query all users with the `creator` role and a completed profile.
    3. Output in valid XML format.
- **acceptance-criteria**:
    - Navigating to `/sitemap.xml` returns a list of all public creator profile links.

### **Task 6: "Inquiry to Client" Conversion Tracking**
- **suggested-assignee**: Fullstack
- **priority**: Medium
- **consideration**: This is internal "Growth Accounting." It helps us see which creators are most successful.
- **description**: Track the conversion rate of a public inquiry becoming a funded milestone.
- **requirements**:
    1. Create a `GrowthStats` table or use an event log.
    2. Record when an inquiry is created via a public profile.
    3. Record when that same project gets its first payment.
- **acceptance-criteria**:
    - Admin can query the database to see the percentage of inquiries that convert to paid clients.

### **Task 7: Simple "Refer a Creator" Reward Logic**
- **suggested-assignee**: Backend
- **priority**: Low
- **consideration**: For V1, just track the referral. Rewards (like 10GB extra storage) can be applied manually.
- **description**: Allow creators to invite other creators and track who brought who into the platform.
- **requirements**:
    1. Add `ReferredByUserID` to the User model.
    2. Update the registration API to accept a `ref` code.
- **acceptance-criteria**:
    - A user registering with `/register?ref=USER_ID` has the referring user's ID linked in their record.

### **Task 8: Implement Vanity URLs for Creators**
- **suggested-assignee**: Senior
- **priority**: Medium
- **consideration**: This requires careful regex validation to prevent users from taking reserved names like "admin," "login," or "api."
- **description**: Allow creators to have URLs like `zeyaroo.com/c/studio-name` instead of UUIDs.
- **requirements**:
    1. Add `slug` (unique string) to `CreatorProfile`.
    2. Update router logic to resolve slugs to user IDs.
    3. Add a "Profile URL" field to settings.
- **acceptance-criteria**:
    - Navigating to `/c/my-studio` loads the correct profile.

### **Task 9: Canonical URL Tags for SEO**
- **suggested-assignee**: Frontend
- **priority**: Low
- **consideration**: Prevents "Duplicate Content" penalties if a profile is accessible via both UUID and Slug.
- **description**: Ensure search engines know which version of a URL is the "master" version.
- **requirements**:
    1. Add `<link rel="canonical" href="..." />` to the head of the Creator Profile page.
- **acceptance-criteria**:
    - Viewing page source shows the canonical link pointing to the vanity URL (if set).

- **suggested-assignee**: Senior
