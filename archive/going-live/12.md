Topic 12: **Creator Productivity (AI & Roadmap)**.
This final module transitions Zeyaroo from a "passive storage" tool to an "active assistant." By automating the tedious parts of the workflow (culling, tagging, archiving), you create high-value features that justify the **Studio Tier** and reduce churn by making the creator faster and more profitable.

### **Task 1: AI Vision API Integration Service**
- **suggested-assignee**: Backend
- **priority**: Medium
- **consideration**: Use a cost-effective provider like Google Vision API or OpenAI (gpt-4o-mini) for initial labeling.
- **description**: Create a service to send media to an AI vision provider to extract descriptive labels and safety ratings.
- **requirements**:
    1. Implement `internal/services/ai_vision.go`.
    2. Function `AnalyzeImage(imagePath) ([]string, error)`.
    3. Trigger this service in the background after thumbnail generation.
- **acceptance-criteria**:
    - Uploaded images have a list of descriptive tags (e.g., "Wedding", "Forest", "Sunset") stored in the `media.metadata` field.

### **Task 2: AI-Powered Duplicate & "Burst" Detection**
- **suggested-assignee**: Senior
- **priority**: Medium
- **consideration**: Use "Perceptual Hashing" (phash) to compare images. This is much cheaper than calling a cloud AI API for every single photo.
- **description**: Identify visually similar photos taken in the same sequence to help creators "cull" (filter) their work faster.
- **requirements**:
    1. Implement a Go library for image hashing.
    2. Group media in a project that have a similarity score > 90%.
    3. Add a `group_id` to the media table.
- **acceptance-criteria**:
    - The API returns groups of similar images, allowing the frontend to collapse them into "stacks."

- **suggested-assignee**: Senior
- **priority**: High
- **consideration**: Moving files to **B2 Cloud Archive** storage can reduce costs by 50-70% for old projects.
- **description**: A service to move a completed project's media to "Cold Storage" after a period of inactivity.
- **requirements**:

- **acceptance-criteria**:
